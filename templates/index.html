01

SQL Injection (SQL 삽입)

사용자 입력 데이터를 검증하지 않고 문자열 포맷팅을 사용해 데이터베이스 쿼리를 생성할 때 발생합니다. 로그인 인증 우회, 데이터 유출 등을 직접 시도해 보세요.

02

XSS (크로스 사이트 스크립팅)

게시글 설명란이나 실시간 채팅창에 입력한 악성 자바스크립트 코드가 필터링 없이 렌더링될 때 발생합니다. 세션 가로채기나 악성 모달 팝업 실행을 확인할 수 있습니다.

03

취약한 비밀번호 암호화

패스워드를 평문으로 저장하거나 MD5 같은 약한 단방향 해시로 처리하는 설계 취약점입니다. 보안 모드 적용 시 강력한 해시 알고리즘(PBKDF2/Bcrypt)으로 교체 및 자동 업그레이드가 진행됩니다.

04

취약한 접근 제어 (IDOR)

현재 사용자가 타인의 글을 삭제하거나 다른 사람의 마이페이지 정보를 강제로 변조할 수 있는 인가 검증 누락 상태를 직접 시도하고, 세션 유효성 검사를 통해 보완합니다.
